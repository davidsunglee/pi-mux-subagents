# Plan: Centralize subagent runtime diagnostics and warning delivery

Source: IDEA-e8f4c2a1
Spec: docs/specs/2026-06-08-e8f4c2a1.md

## Goal

Introduce a single typed-diagnostics abstraction so that warning **delivery** is
selected by runtime context instead of decided locally at each call site. A
producer constructs a typed `Diagnostic` (message + stable `code` + `audience`)
and hands it to one central `emitDiagnostic(diagnostic, context)` dispatcher. The
dispatcher resolves the `human` audience to `ctx.ui.notify(message, "warning")`
when an active UI exists (else stderr), and resolves the `structured` audience to
a per-tool-call collector when one is threaded into the context (else a silent
no-op). The three launch/orchestration warning producers
(`warnGuardedPolicyUnsupported`, `warnClaudeSkillsDropped`,
`warnCodexUnsupportedFeatures`) and the zero-arg `selectBackend()` invalid-mode
warning route through this dispatcher; the tool entry points
(`subagent`, `subagent_run_serial`, `subagent_run_parallel`) create a collector
and surface structured warnings additively in `details.warnings` (per-task for
orchestration). Every existing warning string is preserved byte-for-byte.

## Architecture summary

A new **leaf module** `src/diagnostics/diagnostics.ts` owns the `Diagnostic`
record, the `DiagnosticAudience` set, the `DiagnosticCollector`, the
`DiagnosticContext` (carries the optional collector), and the `emitDiagnostic`
dispatcher. The dispatcher reads the active UI through an **ambient resolver**
registered once by `src/index.ts` as `registerAmbientUi(() => latestCtx)` — this
keeps the existing module-level `latestCtx` as the single source of UI truth and
lets zero-arg callers (`selectBackend()`) join the abstraction with one import.
The module imports nothing from the rest of the codebase, so it cannot create
import cycles (`index.ts → diagnostics`, `index.ts → backends/select.ts →
diagnostics`, `launch/launch-spec.ts → diagnostics`, `backends/headless.ts →
diagnostics` are all acyclic).

The collector is threaded explicitly through the **launch path only** (not watch):
every migrated producer fires synchronously during launch —
`warnGuardedPolicyUnsupported` and `warnClaudeSkillsDropped` /
`warnCodexUnsupportedFeatures` all run before the first `await` of their launch
function (`launchSubagent` for pane; `makeHeadlessBackend.launch` →
`runClaudeHeadless` / `runCodexHeadless` synchronous prefixes for headless), so a
collector handed to launch and drained immediately after `await launch()`
captures all structured warnings. Threading touches: `Backend.launch`
(`src/backends/types.ts`), `LauncherDeps.launch` (`src/orchestration/types.ts`),
`makePaneBackend` / `makeHeadlessBackend`, `makeDefaultDeps`, `launchSubagent`'s
`options`, and `runSerial` / `runParallel` (per-task collector). Collected
warnings ride `OrchestrationResult.warnings` / `OrchestratedTaskResult.warnings`
into `details.results[i].warnings` (orchestration) and into the bare tool's
"started" `details.warnings`.

## Tech stack

- TypeScript (ESM, `.ts` import specifiers), Node.js built-in test runner
  (`node --test`), executed via `pnpm test` / `pnpm typecheck`.
- Peer deps: `@earendil-works/pi-coding-agent` (provides `ExtensionContext`,
  `ExtensionUIContext.notify`), `typebox`. No new dependencies are added.

## Approach (from spec)

The spec's `## Approach` fixes the macro paradigm: **a typed `Diagnostic` record
plus one central dispatcher** (rejecting typed sink objects and typed-callback
formalization). This plan expands that choice into file-level structure; it does
not re-choose the paradigm. The audience taxonomy is `{ human, structured }`; no
`model`-only (structured-without-human) audience is introduced (per Non-Goals).
Open question resolved per spec lean: `details.warnings` is a plain `string[]`;
the stable `code` is retained internally on the `Diagnostic` but **not** exposed
in the tool payload for v1.

  ```ts
  /** Audience set: independently routes the same diagnostic to channels. */
  export interface DiagnosticAudience {
    /** Human channel: UI notify when an active UI exists, else stderr. */
    human?: boolean;
    /** Structured channel: pushed to a tool-call collector when one is present. */
    structured?: boolean;
  }

  /** Stable, internal-only category code (NOT exposed in the tool payload). */
  export type DiagnosticCode =
    | "skills-dropped"
    | "tools-dropped"
    | "guarded-policy-unsupported"
    | "codex-system-prompt-replace"
    | "invalid-subagent-mode";

  export interface Diagnostic {
    code: DiagnosticCode;
    /** Exact human-facing text, including any trailing newline the stderr path expects. */
    message: string;
    audience: DiagnosticAudience;
  }

  /** Minimal UI shape the dispatcher needs; structurally a subset of ExtensionContext. */
  export interface AmbientUiContext {
    hasUI: boolean;
    ui: { notify(message: string, type?: "info" | "warning" | "error"): void };
  }

  /** Per-tool-call structured sink. */
  export interface DiagnosticCollector {
    push(d: Diagnostic): void;
    /** Return collected diagnostic messages byte-for-byte (the exact `message`, including any trailing newline) and clear. */
    drain(): string[];
  }

  /** Context threaded into the dispatcher. Only the structured collector varies per call. */
  export interface DiagnosticContext {
    collector?: DiagnosticCollector;
  }
  ```

- [ ] **Step 2: Implement the collector + ambient resolver + dispatcher.** Append:

  ```ts
  export function createDiagnosticCollector(): DiagnosticCollector {
    const items: Diagnostic[] = [];
    return {
      push(d) { items.push(d); },
      drain() {
        // Byte-for-byte: store and return the diagnostic's exact `message`
        // (including the trailing newline). The spec freezes the producer string
        // and requires the same string mirrored into details.warnings — do NOT trim.
        const out = items.map((d) => d.message);
        items.length = 0;
        return out;
      },
    };
  }

  let ambientUiResolver: () => AmbientUiContext | null = () => null;

  /** Register the live UI source. index.ts passes `() => latestCtx`. */
  export function registerAmbientUi(resolver: () => AmbientUiContext | null): void {
    ambientUiResolver = resolver;
  }

  /**
   * The single delivery decision point. No other site may branch between
   * ui.notify, structured metadata, and process.stderr.write.
   */
  export function emitDiagnostic(diagnostic: Diagnostic, context?: DiagnosticContext): void {
    if (diagnostic.audience.human) {
      const ui = ambientUiResolver();
      if (ui?.hasUI) ui.ui.notify(diagnostic.message.replace(/\n+$/, ""), "warning");
      else process.stderr.write(diagnostic.message);
    }
    if (diagnostic.audience.structured) {
      context?.collector?.push(diagnostic);
    }
  }

  export const __test__ = {
    /** Capture the current resolver so a test can restore it (avoids cross-file contamination). */
    getAmbientUi(): () => AmbientUiContext | null { return ambientUiResolver; },
    setAmbientUi(resolver: () => AmbientUiContext | null): void { ambientUiResolver = resolver; },
  };
  ```

  Note: the human path replicates the old `emitRuntimeWarning` exactly — UI
  notify with trailing newlines stripped, else raw `message` (with newline) to
  stderr. The collector stores the diagnostic's exact `message` byte-for-byte
  (including the trailing newline), so `details.warnings` entries mirror the
  frozen producer string the spec requires — the same string the stderr human
  path writes. (The UI-path newline strip is display-only and does not change the
  canonical structured string.)

- [ ] **Step 3: Write the dispatcher unit test.** Create
  `test/orchestration/diagnostics-dispatch.test.ts`. Capture/restore
  `process.stderr.write` per test, and save/restore the ambient resolver via
  `__test__.getAmbientUi()` / `setAmbientUi()` in a `finally` so no fake UI leaks

  ```ts
  import { describe, it } from "node:test";
  import assert from "node:assert/strict";
  import {
    emitDiagnostic, createDiagnosticCollector, __test__,
    type Diagnostic,
  } from "../../src/diagnostics/diagnostics.ts";

  function withStderr(fn: (read: () => string) => void): void {
    let buf = "";
    const orig = process.stderr.write.bind(process.stderr);
    (process.stderr as any).write = (c: string | Buffer) => { buf += typeof c === "string" ? c : c.toString(); return true; };
    try { fn(() => buf); } finally { (process.stderr as any).write = orig; }
  }

  describe("emitDiagnostic", () => {
    it("human audience with no UI writes raw message (with newline) to stderr", () => {
      const prev = __test__.getAmbientUi();
      __test__.setAmbientUi(() => null);
      try {
        withStderr((read) => {
          emitDiagnostic({ code: "skills-dropped", audience: { human: true }, message: "warn line\n" });
          assert.equal(read(), "warn line\n");
        });
      } finally { __test__.setAmbientUi(prev); }
    });

    it("human audience with active UI notifies (trailing newline stripped) and skips stderr", () => {
      const prev = __test__.getAmbientUi();
      const calls: Array<{ m: string; t?: string }> = [];
      __test__.setAmbientUi(() => ({ hasUI: true, ui: { notify: (m, t) => calls.push({ m, t }) } }));
      try {
        withStderr((read) => {
          emitDiagnostic({ code: "skills-dropped", audience: { human: true }, message: "warn line\n" });
          assert.equal(read(), "");
          assert.deepEqual(calls, [{ m: "warn line", t: "warning" }]);
        });
      } finally { __test__.setAmbientUi(prev); }
    });

    it("structured audience pushes the exact message (byte-for-byte) to the collector; no collector is a silent no-op", () => {
      const prev = __test__.getAmbientUi();
      __test__.setAmbientUi(() => null);
      try {
        withStderr(() => {
          const collector = createDiagnosticCollector();
          const d: Diagnostic = { code: "tools-dropped", audience: { structured: true }, message: "structured line\n" };
          emitDiagnostic(d, { collector });
          assert.deepEqual(collector.drain(), ["structured line\n"]); // byte-for-byte: trailing newline preserved
          assert.deepEqual(collector.drain(), []); // drain clears
          assert.doesNotThrow(() => emitDiagnostic(d)); // no collector -> no throw
        });
      } finally { __test__.setAmbientUi(prev); }
    });

    it("{human,structured} with no UI delivers stderr AND collects; new sites need no local branching", () => {
      const prev = __test__.getAmbientUi();
      __test__.setAmbientUi(() => null);
      try {
        withStderr((read) => {
          const collector = createDiagnosticCollector();
          emitDiagnostic({ code: "guarded-policy-unsupported", audience: { human: true, structured: true }, message: "both line\n" }, { collector });
          assert.equal(read(), "both line\n");
          assert.deepEqual(collector.drain(), ["both line\n"]); // byte-for-byte: same string as the stderr human path
        });
      } finally { __test__.setAmbientUi(prev); }
    });
  });
  ```

## Risk Assessment

- **Headless human-audience warnings now UI-route when the parent TUI is active.**
  Before this change, `warnClaudeSkillsDropped` / `warnCodexUnsupportedFeatures` /
  `warnGuardedPolicyUnsupported` on the headless path always wrote stderr. After
  migration they route through the dispatcher, so when the parent pi process has
  an active TUI (`latestCtx.hasUI`), they deliver via `ui.notify`. This is the
  intended centralization (don't corrupt the active editor) and is consistent
  with the spec's "context-aware human-audience delivery". The headless
  integration regression (`test/integration/headless-claude-skills-warning.test.ts`)
  never sets a UI, so it still observes stderr — no test breakage. Mitigation:
  documented here; the non-migrated child-runner diagnostics stay raw stderr.
- **Cross-file ambient-resolver contamination in the shared test process.**
  `pnpm test` runs all files in one `node --test` process, so the diagnostics
  module's `ambientUiResolver` is shared state. Mitigation: the `__test__` seam is
  save/restore (`getAmbientUi` → `setAmbientUi(prev)`), and every test that swaps
  in a fake UI restores the prior resolver in a `finally`. The pane UI test uses
  `latestCtx` via `session_start` / `session_shutdown` (not the seam) and resets
  in `finally`.
- **Import-cycle risk.** `select.ts`, `launch-spec.ts`, `headless.ts`, and
  `index.ts` all import the diagnostics module. Mitigation: the diagnostics module
  is a strict leaf (imports nothing from the project) and reads the UI through a
  resolver function registered by `index.ts`, so `latestCtx` stays the single
  source of truth without the module importing `index.ts`.
- **Collector timing for the bare headless `subagent` result.** Structured
  warnings are surfaced only if they fire before `await backend.launch()` resolves.
  Verified: `warnGuardedPolicyUnsupported` runs in `headless.launch` directly, and
  `warnClaudeSkillsDropped` / `warnCodexUnsupportedFeatures` run in the
  `runClaudeHeadless` / `runCodexHeadless` synchronous prefixes (before their first
  `await`), which execute when the run promise is constructed inside `launch`.
  Watch-time diagnostics are out of scope (none of the migrated producers fire
  during watch).
- **`runParallel` blocked-task `continue` path.** A task that blocks (pi
  `caller_ping`) hits `continue` before terminalization, so its launch warnings
  are not attached to a terminal slot. This is acceptable: blocked is a mid-run,
  non-terminal state and the registry owns its lifecycle; warnings attach on the
  eventual terminal/resume result. Noted, not mitigated.
- **Optional-param compatibility.** Adding a trailing optional `diagnostics` param
  to `Backend.launch` and `LauncherDeps.launch` is backward compatible — existing
  implementers and test stubs that accept fewer params remain assignable in
  TypeScript. Verified by the Task 3 / Task 7 `pnpm typecheck` gates.
